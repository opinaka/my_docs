📌 1. TypeDoc (L’équivalent de JSDoc pour TypeScript)

📌 TypeDoc est l’outil recommandé pour générer une documentation interactive et automatique à partir du code TypeScript.

🔹 Avantages
✅ Génération automatique à partir des types TypeScript
✅ Génère une doc HTML interactive
✅ Permet une meilleure structuration que JSDoc

🔧 Installation & Utilisation

  1. Installer TypeDoc

    npm install --save-dev typedoc

  2. Ajouter des commentaires TypeDoc

    /**
 * Additionne deux nombres.
 * @param a Le premier nombre
 * @param b Le second nombre
 * @returns La somme de a et b
 */
function add(a: number, b: number): number {
    return a + b;
}

  3. Générer la documentation

    npx typedoc --out docs src/

📎 Site Web : https://typedoc.org

📝 2. JSDoc (L’outil standard pour JavaScript)

📌 JSDoc est l’outil le plus populaire pour documenter du JavaScript. Il permet d’ajouter des annotations et de générer une documentation HTML.

🔹 Avantages
✅ Compatible avec JavaScript et TypeScript
✅ Génération automatique en HTML
✅ Facile à intégrer dans un projet

🔧 Installation & Utilisation

  1. Installer JSDoc

    npm install --save-dev jsdoc

  2. Ajouter des commentaires JSDoc

    /**
 * Additionne deux nombres.
 * @param {number} a Le premier nombre
 * @param {number} b Le second nombre
 * @returns {number} La somme de a et b
 */
function add(a, b) {
    return a + b;
}

  3. Générer la documentation

    npx jsdoc -c jsdoc.json

📎 Site Web : https://jsdoc.app

🔍 3. ESDoc (Une alternative plus moderne à JSDoc)

📌 ESDoc est une alternative à JSDoc, offrant plus de fonctionnalités et une meilleure prise en charge d’ES6+.

🔹 Avantages
✅ Plus moderne que JSDoc
✅ Supporte les décorateurs et les classes ES6
✅ Documentation plus détaillée

🔧 Installation & Utilisation

  1. Installer ESDoc

    npm install --save-dev esdoc

  2. Ajouter des commentaires

    /**
 * Classe représentant une personne.
 */
class Person {
    /**
     * Crée une nouvelle personne.
     * @param {string} name Le nom de la personne.
     */
    constructor(name) {
        this.name = name;
    }
}

  3. Générer la documentation

    npx esdoc

📎 Site Web : https://esdoc.org

📚 4. Docusaurus (Documentation interactive type Wiki)

📌 Docusaurus est un framework de documentation basé sur React, utilisé par Facebook.

🔹 Avantages
✅ Utilisation facile avec Markdown
✅ Génération d’un site web interactif
✅ Compatible avec JavaScript et TypeScript

🔧 Installation & Utilisation

  1. Installer Docusaurus

    npx create-docusaurus@latest my-docs classic

  2. Ajouter du contenu Markdown

    • Modifier les fichiers .md dans docs/

  3. Lancer le serveur local

    npm run start

📎 Site Web : https://docusaurus.io

🏗️ 5. MkDocs (Documentation en Markdown)

📌 MkDocs est une alternative simple à Docusaurus, qui utilise Markdown au lieu de React.

🔹 Avantages
✅ Très facile à configurer
✅ Génère un site web statique
✅ Compatible avec TypeScript

🔧 Installation & Utilisation

  1. Installer MkDocs

    pip install mkdocs

  2. Créer une documentation

    mkdocs new my_project

  3. Lancer le serveur local

    mkdocs serve

📎 Site Web : https://www.mkdocs.org

🔥 6. Swagger / OpenAPI (Documentation des API en TypeScript)

📌 Swagger (OpenAPI) est un standard pour documenter les API REST en TypeScript.

🔹 Avantages
✅ Génération automatique de documentation API
✅ Interface interactive avec tests d’API
✅ Compatible avec NestJS, Express.js et Fastify

🔧 Installation & Utilisation avec NestJS

  1. Installer Swagger pour NestJS

    npm install --save @nestjs/swagger swagger-ui-express

  2. Ajouter Swagger dans main.ts

    import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';

const config = new DocumentBuilder()
  .setTitle('API Exemple')
  .setDescription('Documentation de l’API')
  .setVersion('1.0')
  .build();
const document = SwaggerModule.createDocument(app, config);
SwaggerModule.setup('api', app, document);

📎 Site Web : https://swagger.io
