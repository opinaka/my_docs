MkDocs pour la documentation de projets en différents langages de programmation

MkDocs est un générateur de site statique conçu pour créer de la documentation de projet. Voici un guide complet pour l'utiliser avec différents langages de programmation.

Installation de MkDocs

pip install mkdocs

Pour des fonctionnalités supplémentaires, installez le thème Material (thème) qui est très populaire :

pip install mkdocs-material

Structure de base d'un projet MkDocs

mon-projet/
├── mkdocs.yml    # Fichier de configuration
└── docs/
    ├── index.md  # Page d'accueil
    └── api.md    # Documentation d'API

Configuration de base (mkdocs.yml)

site_name: Documentation de Mon Projet
theme: material

nav:
  - Accueil: index.md
  - Guide utilisateur: guide.md
  - Documentation API: api.md
  - À propos: about.md

Commandes MkDocs essentielles

# Créer un nouveau projet
mkdocs new mon-projet

# Serveur local de développement
mkdocs serve

# Construire la documentation
mkdocs build

# Déployer sur GitHub Pages
mkdocs gh-deploy

Documentation pour différents langages de programmation

Python

Pour Python, ajoutez mkdocstrings pour générer automatiquement de la documentation à partir des docstrings :

pip install mkdocstrings[python]

Dans mkdocs.yml :

plugins:
  - search
  - mkdocstrings:
      handlers:
        python:
          selection:
            docstring_style: google

Dans votre fichier Markdown :

# API Documentation

::: mon_module.ma_classe
    handler: python
    selection:
      members:
        - methode1
        - methode2

JavaScript/TypeScript

Pour JavaScript/TypeScript, utilisez typedoc et typedoc-plugin-markdown :

npm install --save-dev typedoc typedoc-plugin-markdown

Ajoutez un script dans votre package.json :

"scripts": {
  "docs": "typedoc --out docs/api src/ --plugin typedoc-plugin-markdown"
}

Java

Pour Java, vous pouvez utiliser Javadoc avec mkdocs-javadoc-plugin :

pip install mkdocs-javadoc-plugin

Dans mkdocs.yml :

plugins:
  - search
  - javadoc:
      source_dir: src/main/java
      output_dir: docs/javadoc

C/C++

Pour C/C++, utilisez Doxygen pour générer la documentation puis intégrez-la avec mkdocs-doxygen-plugin :

pip install mkdocs-doxygen-plugin

Dans mkdocs.yml :

plugins:
  - search
  - doxygen:
      doxygen_dir: doxygen

Déploiement

GitHub Pages

mkdocs gh-deploy

GitLab Pages

Créez un fichier .gitlab-ci.yml :

pages:
  stage: deploy
  script:
    - pip install mkdocs mkdocs-material
    - mkdocs build
  artifacts:
    paths:
      - site
  only:
    - main

Conseils pour une documentation efficace

  1. Utilisez une structure cohérente avec des sections claires
  2. Incluez des exemples de code pour chaque fonctionnalité
  3. Ajoutez des captures d'écran ou des diagrammes si nécessaire
  4. Créez des guides pas à pas pour les opérations complexes
  5. Maintenez votre documentation à jour avec votre code
