📌 1. Sphinx (L’équivalent de Swagger pour Python)

📌 Sphinx est l’outil le plus utilisé pour générer une documentation professionnelle et interactives en Python.

🔹 Avantages
✅ Génération automatique depuis les docstrings
✅ Compatible avec Markdown et reStructuredText
✅ Utilisé pour la documentation de Django, Flask et NumPy

🔧 Installation & Utilisation

  1. Installer Sphinx

    pip install sphinx

  2. Créer une documentation

    sphinx-quickstart

  3. Générer la doc

    make html

📎 Site Web : https://www.sphinx-doc.org

📝 2. MkDocs (Documentation en Markdown)

📌 MkDocs est une alternative simple à Sphinx, qui utilise Markdown au lieu de reStructuredText.

🔹 Avantages
✅ Très facile à configurer
✅ Génère un site web statique
✅ Utilisé pour des docs de projets open-source

🔧 Installation & Utilisation

  1. Installer MkDocs

    pip install mkdocs

  2. Créer une documentation

    mkdocs new my_project

  3. Lancer le serveur local

    mkdocs serve

📎 Site Web : https://www.mkdocs.org

🔍 3. pydoc (Documentation intégrée de Python)

📌 pydoc est inclus dans Python et permet d'afficher la documentation des modules.

🔹 Avantages
✅ Aucune installation requise
✅ Génération HTML et CLI
✅ Compatible avec toutes les versions de Python

🔧 Exemples d'utilisation

  • Afficher la documentation d'un module : 
    pydoc math
  • Générer une doc HTML d’un module : 
    pydoc -w mymodule

📎 Documentation officielle : https://docs.python.org/3/library/pydoc.html

📚 4. Doxygen (Documentation de Code avec Graphiques)

📌 Doxygen est un outil puissant permettant de documenter du code source en générant des diagrammes UML et des liens entre les fonctions.

🔹 Avantages
✅ Génération HTML et PDF
✅ Supporte Python, C, C++, Java
✅ Intégration avec Graphviz pour les schémas

🔧 Installation & Utilisation

  1. Installer Doxygen

    sudo apt install doxygen

  2. Générer un fichier de config :

    doxygen -g Doxyfile

  3. Exécuter :

    doxygen Doxyfile

📎 Site Web : https://www.doxygen.nl

✍ 5. Utiliser les Docstrings avec Sphinx-Autodoc

📌 Autodoc est un module de Sphinx qui extrait la documentation directement depuis les docstrings.

🔹 Exemple de Docstring au format reStructuredText

def add(a: int, b: int) -> int:
    """
    Additionne deux nombres.

    :param a: Le premier nombre
    :param b: Le second nombre
    :return: La somme de a et b
    """
    return a + b

🔧 Générer la documentation avec Sphinx

sphinx-apidoc -o docs/ myproject/

📎 Plus d'infos : https://www.sphinx-doc.org/en/master/usage/extensions/autodoc.html

🖥️ 6. Pycco (Documentation type "Literate Programming")

📌 Pycco est un outil qui génère une documentation côte à côte avec le code source.

🔹 Avantages
✅ Génère des pages HTML claires
✅ Format inspiré de Literate Programming
✅ Simple et rapide

🔧 Installation & Utilisation

  1. Installer Pycco

    pip install pycco

  2. Générer la documentation

    pycco mon_script.py

📎 Site Web : https://pycco-docs.github.io

🎯 Conclusion : Quel outil choisir ?

Besoin Outil recommandé
🔹 Doc interactive (Swagger-like) Sphinx
🔹 Doc simple en Markdown MkDocs

🔹 Documentation technique + Graphiques Doxygen
🔹 Docstrings + Génération auto Sphinx-Autodoc
