Générer un site web à partir de tes fichiers Markdown

Si tu utilises Obsidian pour rédiger ta documentation technique et que tu souhaites générer un site web à partir de tes fichiers Markdown, voici les principaux outils qui peuvent t'aider :

1️⃣ Obsidian Publish (solution native)

📌 Obsidian Publish est la solution officielle d'Obsidian permettant de publier directement tes notes sous forme de site web.
🔹 Avantages :

  • Hébergement simple sans besoin de serveur.
  • Interface propre et rapide.
  • Gestion des liens internes et des graphes interactifs.
    🔸 Inconvénients :
  • Service payant (8 $/mois).
  • Peu de personnalisation avancée.

💡 Idéal si tu veux une solution clé en main sans te soucier de l’hébergement.

2️⃣ MkDocs (avec Material for MkDocs)

📌 MkDocs est un générateur de site statique basé sur Markdown, très utilisé pour la documentation technique.
🔹 Avantages :

  • Facile à configurer.
  • Très rapide et léger.
  • Material for MkDocs offre un design moderne et interactif.
  • Peut être hébergé gratuitement sur GitHub Pages ou Netlify.

🔧 Installation & Utilisation

pip install mkdocs-material
mkdocs new my-docs
cd my-docs
mkdocs serve  # Lance un serveur local
mkdocs build  # Génère les fichiers HTML

💡 Idéal si tu veux un site rapide et professionnel sans dépendre d’un service payant.

3️⃣ Docusaurus (basé sur React)

📌 Docusaurus est un framework développé par Meta pour créer des documentations techniques.
🔹 Avantages :

  • Très personnalisable avec React.
  • Gestion facile du Markdown.
  • Peut être hébergé gratuitement (GitHub Pages, Vercel, Netlify).
  • Bonne intégration avec Obsidian grâce à un plugin Markdown.

🔧 Installation & Utilisation

npx create-docusaurus@latest my-site classic
cd my-site
npm start  # Lance un serveur local
npm run build  # Génère les fichiers HTML

💡 Idéal si tu veux un site moderne avec des composants React personnalisables.

4️⃣ Hugo (pour générer un site ultra-rapide)

📌 Hugo est un générateur de site statique très rapide écrit en Go.
🔹 Avantages :

  • Très performant (charge rapide).
  • Nombreux thèmes Markdown adaptés à la documentation.
  • Peut être hébergé sur GitHub Pages, Netlify, Vercel.

🔧 Installation & Utilisation

brew install hugo  # macOS
choco install hugo  # Windows
hugo new site my-docs
cd my-docs
hugo server  # Serveur local
hugo -D  # Générer le site

💡 Idéal si tu veux un site ultra-rapide et bien structuré.

5️⃣ Quartz (Spécifique pour Obsidian)

📌 Quartz est une solution open-source conçue spécialement pour convertir Obsidian en site web statique.
🔹 Avantages :

  • Compatible avec les graphes d’Obsidian.
  • Personnalisation possible avec CSS.
  • Open-source et gratuit.

🔧 Installation & Utilisation

  1. Clone le dépôt Quartz :

    git clone https://github.com/jackyzha0/quartz.git my-obsidian-site

  2. Copie tes fichiers Markdown d’Obsidian dans le dossier content.

  3. Lance Quartz :

    cd my-obsidian-site
npm install
npm run dev

💡 Idéal si tu veux une solution dédiée à Obsidian avec une navigation fluide.

6️⃣ Obsidian Digital Garden (GitHub Pages)

📌 Digital Garden est un template permettant de convertir tes fichiers Obsidian en site web avec GitHub Pages.
🔹 Avantages :

  • Facile à déployer sur GitHub Pages.
  • Interface propre et légère.
  • Supporte les backlinks et graphes interactifs.

🔧 Installation & Déploiement

  1. Forker ce repo : Obsidian Digital Garden.
  2. Modifier le contenu et push sur GitHub.
  3. Activer GitHub Pages pour le dépôt.

💡 Idéal si tu veux publier tes notes d’Obsidian gratuitement sur GitHub Pages.

🔹 Conclusion

  • ✅ Si tu veux une solution simple et rapide, Obsidian Publish est idéal (mais payant).
  • 🚀 Si tu veux une documentation technique propre et rapide, MkDocs + Material for MkDocs est la meilleure option.
  • 🔥 Si tu veux un site très personnalisable avec React, Docusaurus est un bon choix.
  • ⚡ Si la performance est ta priorité, Hugo est parfait.
  • 📚 Si tu veux garder l’expérience Obsidian sur ton site, Quartz ou Obsidian Digital Garden sont faits pour toi.
