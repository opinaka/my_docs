00.0 Documentation avec Mkdocs

Voici un tutoriel complet sur MkDocs, un générateur de documentation statique simple et efficace basé sur Markdown et Python.

📂 1. Projet Existant : Catch2

Catch2 est un Framework moderne de tests unitaires pour le langage C++.

 git clone https://github.com/catchorg/catch2.git

Le répertoire docs du dépôt Catch2 contient plusieurs fichiers de documentation au format Markdown (.md). Ces fichiers fournissent une documentation complète pour comprendre et utiliser efficacement Catch2 dans vos projets C++.

 ls catch2/docs

Ajoutez le fichier mkdocs.yml dans le dossier :

touch mkdocs.yml

# nano mkdocs.yml
site_name: Catch2
docs_dir: Catch2/docs
theme: readthedocs

Lancez le serveur local pour voir la documentation :

mkdocs serve

Accédez à http://127.0.0.1:8000/ pour voir la documentation.

Pour générer la documentation en fichiers statiques :

mkdocs build

📂 2. Création d’un nouveau projet MkDocs

Générez un nouveau projet :

mkdocs new my-docs
cd my-docs

Le projet contient ces fichiers :

my-docs/
│── docs/
│   ├── index.md  # Page d'accueil
│── mkdocs.yml  # Configuration du site

Lancez le serveur local pour voir la documentation :

mkdocs serve

Accédez à http://127.0.0.1:8000/ pour voir la documentation.

✍ Ajout de contenu

Ajoutez des fichiers Markdown dans le dossier docs/ :

echo "# Introduction" > docs/intro.md
echo "# Installation" > docs/install.md

Modifiez mkdocs.yml pour définir la structure :

site_name: Ma Documentation
nav:
  - Accueil: index.md
  - Introduction: intro.md
  - Installation: install.md

Rechargez la page du serveur pour voir les modifications.

🎨 Personnalisation avec des thèmes

Utiliser le thème Material for MkDocs

pip install mkdocs-material

Modifiez mkdocs.yml :

theme:
  name: material

Relancez le serveur :

mkdocs serve

📦 Génération et déploiement

Pour générer la documentation en fichiers statiques :

mkdocs build

Cela crée un dossier site/ contenant les fichiers HTML.

Pour déployer sur GitHub Pages :

mkdocs gh-deploy

Il faut avoir un repo GitHub configuré pour que cela fonctionne.

🛠 3. Installation de Python et création d’un environnement virtuel

Avant d’utiliser MkDocs, assurez-vous que Python est installé sur votre machine.

🔹 Installation de Python

Si Python n'est pas encore installé :

  • Windows : Téléchargez et installez Python depuis python.org.
  • Linux/macOS : Installez avec : 
    sudo apt install python3  # Debian/Ubuntu
sudo dnf install python3  # Fedora
brew install python       # macOS

🔹 Création d’un environnement virtuel

Il est recommandé d’utiliser un environnement virtuel pour isoler les dépendances du projet :

python3 -m venv venv
source venv/bin/activate  # Sur macOS/Linux
venv\Scripts\activate     # Sur Windows

🚀 Installation de MkDocs

Avec l’environnement activé, installez MkDocs via pip :

pip install mkdocs

Vérifiez l’installation :

mkdocs --version

Combiner Obsidian avec MkDocs

Vous pouvez copier manuellement vos fichiers Markdown depuis votre vault Obsidian vers le répertoire docs/ de votre projet MkDocs. Assurez-vous que la structure des dossiers et les liens internes sont conformes aux exigences de MkDocs.

Obsidian introduit certaines extensions spécifiques, telles que les liens internes de type WikiLinks, qui peuvent ne pas être directement compatibles avec tous les générateurs de sites statiques comme MkDocs.

Les WikiLinks sont une extension du langage Markdown qui permet de créer des liens internes de manière simplifiée, en entourant le nom de la page cible avec des doubles crochets [[ ]]. Cette syntaxe est particulièrement utile pour les wikis et les systèmes de documentation, facilitant la liaison entre différentes pages sans avoir à spécifier explicitement les URLs.

Exemple de document Markdown utilisant des WikiLinks :

  • [[Introduction]] pointe vers une page intitulée "Introduction".
  • [[Guide d'utilisation]] pointe vers une page intitulée "Guide d'utilisation".
  • [[FAQ]] pointe vers une page intitulée "FAQ".
  • [[Contact]] pointe vers une page intitulée "Contact".

Pour adapter vos liens internes au format compatible avec MkDocs, il est recommandé d'utiliser la syntaxe standard de Markdown en spécifiant le chemin relatif vers le fichier Markdown cible. Voici comment procéder :

  1. Identifier le fichier cible : Déterminez le nom du fichier Markdown vers lequel vous souhaitez créer un lien.

  2. Utiliser la syntaxe de lien Markdown : Employez la syntaxe suivante pour créer le lien :

    [Texte du lien](chemin/vers/fichier.md)

    Par exemple, pour lier vers un fichier guide_utilisation.md situé dans le même répertoire, utilisez :

    [Guide d'utilisation](guide_utilisation.md)

  3. Liens vers des sections spécifiques : Pour lier vers une section spécifique d'un document, ajoutez un identifiant de section à l'URL. MkDocs génère automatiquement des ancres pour chaque titre de section. Par exemple, pour lier vers une section intitulée "Installation" dans guide_utilisation.md, utilisez :

    [Installation](guide_utilisation.md#installation)

Remarques importantes :

  • Structure des répertoires : Assurez-vous que la structure des répertoires est correctement reflétée dans vos chemins relatifs. Par exemple, si guide_utilisation.md se trouve dans un sous-dossier docs/, le lien serait : 
    [Guide d'utilisation](docs/guide_utilisation.md)
  • Compatibilité MkDocs : MkDocs prend en charge cette syntaxe de liens internes. Lors de la génération du site, les liens seront automatiquement convertis en URLs appropriées.

En appliquant cette méthode, vos liens internes seront compatibles avec MkDocs, garantissant une navigation fluide au sein de votre documentation.

Déployer manuellement votre site web statique sur GitHub

Pour déployer manuellement votre site web statique sur GitHub sans utiliser Git, vous pouvez télécharger directement vos fichiers dans votre dépôt GitHub via l'interface web. Voici comment procéder :

  1. Créer un dépôt GitHub :

    • Connectez-vous à votre compte GitHub.
    • Cliquez sur le bouton "New" pour créer un nouveau dépôt.
    • Nommez le dépôt selon la convention votre_nom_utilisateur.github.io pour un site web personnel, ou choisissez un autre nom pour un projet spécifique.
    • Cliquez sur "Create repository".

  2. Accéder au dépôt via l'interface web :

    • Une fois le dépôt créé, GitHub vous proposera des instructions pour ajouter des fichiers.
    • Cliquez sur "uploading an existing file" ou sur "Add file" puis "Upload files".

  3. Télécharger vos fichiers :

    • Sur la page d'upload, faites glisser vos fichiers et dossiers depuis votre ordinateur vers la zone de dépôt sur GitHub.
    • Assurez-vous que la structure de vos dossiers est conforme à celle souhaitée pour votre site.

  4. Valider l'ajout des fichiers :

    • Après avoir ajouté vos fichiers, descendez à la section "Commit changes".
    • Ajoutez un message de commit, par exemple : "Ajout des fichiers du site web".
    • Cliquez sur "Commit changes" pour enregistrer les modifications.

  5. Activer GitHub Pages :

    • Dans votre dépôt, cliquez sur l'onglet "Settings" (Paramètres).
    • Dans la section "Pages", sous "Source", sélectionnez la branche main et le répertoire / (root).
    • Cliquez sur "Save" pour appliquer les modifications.

  6. Accéder à votre site :

    • Après quelques minutes, votre site sera accessible à l'adresse https://votre_nom_utilisateur.github.io.
