Cours sur l'utilisation de Read the Docs pour la documentation technique

Introduction

Read the Docs est une plateforme qui facilite l'hébergement et la gestion de la documentation technique. Elle est particulièrement utilisée pour les projets open source, mais elle peut également être bénéfique pour les projets privés. Ce cours vous guidera à travers les étapes nécessaires pour créer, héberger et maintenir votre documentation avec Read the Docs.

Objectifs du cours

  • Comprendre ce qu'est Read the Docs et ses avantages.
  • Apprendre à configurer un projet sur Read the Docs.
  • Savoir comment rédiger et organiser la documentation.
  • Découvrir comment déployer et mettre à jour la documentation.

1. Qu'est-ce que Read the Docs ?

Read the Docs permet de créer une documentation facilement accessible et bien organisée. Il propose plusieurs fonctionnalités :

  • Hébergement gratuit pour les projets open source.
  • Support de Sphinx, un générateur de documentation basé sur Python.
  • Versions multiples de la documentation.
  • Intégration avec des systèmes de contrôle de version comme Git.

2. Configuration d'un projet sur Read the Docs

2.1. Création d'un compte

  1. Rendez-vous sur Read the Docs.
  2. Cliquez sur "Sign up" pour créer un compte.
  3. Connectez-vous avec votre compte GitHub, GitLab ou Bitbucket.

2.2. Importation d'un projet

  1. Une fois connecté, cliquez sur "Import a Project".
  2. Choisissez le dépôt que vous souhaitez importer (assurez-vous que le dépôt est public ou que vous avez les autorisations nécessaires pour un dépôt privé).
  3. Configurez les paramètres du projet (nom, langue, etc.).

2.3. Configuration du fichier readthedocs.yml

Créez un fichier nommé readthedocs.yml à la racine de votre dépôt. Ce fichier contient des configurations pour le build de la documentation. Exemple de contenu :

version: 2

sphinx:
  configuration: docs/conf.py

python:
  install:
    - requirements: requirements.txt

3. Rédaction et organisation de la documentation

3.1. Utilisation de Sphinx

Sphinx est un générateur de documentation qui utilise reStructuredText ou Markdown. Pour commencer :

  1. Installez Sphinx dans votre environnement de développement : 
    pip install sphinx
  2. Initialisez Sphinx dans le dossier docs de votre projet : 
    sphinx-quickstart docs

3.2. Structure de la documentation

Organisez votre documentation en plusieurs fichiers et dossiers. Un exemple de structure :

docs/
    conf.py
    index.rst
    installation.rst
    usage.rst
    api/
        api_reference.rst

3.3. Rédaction des fichiers

Utilisez la syntaxe reStructuredText ou Markdown pour rédiger vos fichiers. Voici un exemple d'un fichier index.rst :

.. toctree::
   :maxdepth: 2

   installation
   usage
   api/api_reference

4. Déploiement et mise à jour de la documentation

4.1. Build de la documentation

Chaque fois que vous poussez une mise à jour sur votre dépôt, Read the Docs déclenche automatiquement un build. Vous pouvez vérifier l'état de votre build sur la page de votre projet.

4.2. Accéder à la documentation

Une fois le build terminé, vous pouvez accéder à votre documentation à l'URL fournie par Read the Docs, généralement sous la forme https://votre-projet.readthedocs.io.

4.3. Mise à jour de la documentation

Pour mettre à jour la documentation :

  1. Modifiez les fichiers .rst ou .md de votre projet.
  2. Commitez vos modifications et poussez-les sur votre dépôt.
  3. Le build sera automatiquement déclenché et votre documentation mise à jour.

Conclusion

Read the Docs offre une solution puissante pour héberger et gérer la documentation technique. En suivant ce cours, vous devriez être en mesure de créer, déployer et maintenir votre documentation avec succès. N'hésitez pas à explorer les fonctionnalités avancées de RTD, comme les thèmes personnalisés et les intégrations.
