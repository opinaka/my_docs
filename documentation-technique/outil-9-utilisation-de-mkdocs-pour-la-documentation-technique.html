Cours sur l'utilisation de MkDocs pour la documentation technique

Introduction

MkDocs est un générateur de documentation statique, conçu pour rendre la création de documentation simple et efficace. Il utilise Markdown pour le contenu et peut générer des sites web élégants et responsives. Ce cours vous guidera à travers le processus d'installation, de configuration, de rédaction et de publication de votre documentation technique avec MkDocs.

Objectifs du cours

  • Comprendre les fonctionnalités de MkDocs.
  • Installer MkDocs sur votre machine.
  • Créer un projet MkDocs.
  • Rédiger et organiser la documentation en utilisant Markdown.
  • Publier la documentation sur un serveur local ou en ligne.

1. Qu'est-ce que MkDocs ?

MkDocs est un outil open source qui permet de créer facilement des sites de documentation. Ses principales caractéristiques incluent :

  • Utilisation de Markdown : écrivez votre documentation avec une syntaxe simple et lisible.
  • Thèmes personnalisables : choisissez parmi plusieurs thèmes pour personnaliser l'apparence de votre documentation.
  • Intégration facile : déployez votre documentation sur des plateformes comme GitHub Pages.

2. Installation de MkDocs

2.1. Prérequis

Avant d'installer MkDocs, assurez-vous d'avoir :

  • Python (version 3.6 ou supérieure) installé sur votre machine.
  • Pip (le gestionnaire de paquets Python) installé.

2.2. Installation via pip

Ouvrez un terminal et exécutez la commande suivante :

pip install mkdocs

Vous pouvez vérifier si MkDocs est installé correctement en exécutant :

mkdocs --version

3. Création d'un projet MkDocs

3.1. Initialisation du projet

Pour créer un nouveau projet MkDocs, exécutez la commande suivante dans le terminal :

mkdocs new mon_projet

Cela créera un dossier nommé mon_projet contenant la structure de fichiers de base.

3.2. Structure d'un projet MkDocs

La structure de votre projet ressemblera à ceci :

mon_projet/
    mkdocs.yml
    docs/
        index.md
  • mkdocs.yml : fichier de configuration principal de votre projet.
  • docs/ : dossier où vous placez tous vos fichiers Markdown.

4. Configuration du projet

4.1. Modifier le fichier mkdocs.yml

Ouvrez mkdocs.yml dans un éditeur de texte. Vous pouvez modifier les paramètres suivants :

  • Titre : le titre de votre documentation.
  • Thèmes : choisissez un thème pour votre documentation (par défaut, MkDocs utilise le thème mkdocs).

Exemple de configuration de mkdocs.yml :

site_name: Mon Projet
theme:
  name: material

4.2. Ajout de pages

Ajoutez des fichiers Markdown dans le dossier docs/ pour chaque section de votre documentation. Par exemple :

touch docs/installation.md
touch docs/utilisation.md

Ajoutez ensuite ces fichiers à votre configuration mkdocs.yml :

nav:
  - Accueil: index.md
  - Installation: installation.md
  - Utilisation: utilisation.md

5. Rédaction de la documentation

5.1. Utilisation de Markdown

Rédigez votre documentation en utilisant Markdown. Voici quelques éléments de base :

  • Titres : # Titre 1, ## Titre 2, ...

  • Listes :

    • Liste à puces : 
      - Élément 1
- Élément 2
    • Liste numérotée : 
      1. Premier élément
2. Deuxième élément

  • Liens : [Texte du lien](URL)

  • Images : ![Texte alternatif](URL de l'image)

5.2. Exemple de contenu

Voici un exemple de contenu pour index.md :

# Bienvenue dans ma documentation

Cette documentation vous aidera à comprendre comment utiliser mon projet.

## Table des matières

- [Installation](installation.md)
- [Utilisation](utilisation.md)

6. Visualisation de la documentation

6.1. Serveur local

Pour visualiser votre documentation localement, exécutez la commande suivante dans le répertoire de votre projet :

mkdocs serve

Cela lancera un serveur local. Vous pouvez accéder à votre documentation en naviguant à l'adresse http://127.0.0.1:8000 dans votre navigateur.

7. Publication de la documentation

7.1. Génération des fichiers statiques

Pour générer les fichiers statiques de votre documentation, exécutez :

mkdocs build

Cela créera un dossier site/ contenant les fichiers HTML.

7.2. Déploiement sur GitHub Pages

Pour déployer votre documentation sur GitHub Pages, exécutez :

mkdocs gh-deploy

Assurez-vous que votre projet est initialisé avec Git et qu'il est connecté à un dépôt GitHub.

Conclusion

MkDocs est un outil puissant et facile à utiliser pour créer de la documentation technique. En suivant ce cours, vous devriez être en mesure de configurer, rédiger, visualiser et publier votre documentation efficacement. N'hésitez pas à explorer les fonctionnalités avancées de MkDocs, telles que l'ajout d'extensions et la personnalisation des thèmes.
