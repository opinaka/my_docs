Obsidian

Oui, je connais Obsidian ! C'est un outil de prise de notes et de gestion des connaissances basé sur du Markdown. Il est très populaire pour la prise de notes liée grâce à son système de graphes qui permet de visualiser les connexions entre les notes.

Obsidian est souvent utilisé pour :

  • La gestion de projets et d'idées (Zettelkasten, PKM - Personal Knowledge Management).
  • La documentation technique grâce à la prise en charge de Markdown, de LaTeX et de code blocks.
  • L'organisation des recherches (scientifiques, académiques, ou personnelles).
  • L'écriture structurée (articles, livres, documentation).

Il est offline-first (les notes sont stockées localement) mais peut être synchronisé via Obsidian Sync ou d'autres solutions comme Git ou Syncthing.

Tu l’utilises déjà ou tu cherches à l’explorer pour un usage particulier ? 😊

Oui, Obsidian permet de publier directement un projet sous forme de site web grâce à plusieurs solutions ! Voici les principales méthodes pour transformer tes notes Obsidian en un site web accessible en ligne :

✅ 1. Obsidian Publish (Solution Officielle - Payante)

🚀 Simple et rapide, mais payant
Obsidian propose un service intégré, Obsidian Publish, qui permet de publier directement tes notes sur un site web hébergé par Obsidian.

Avantages :
✔ Facile à configurer directement depuis l’interface Obsidian.
✔ Synchronisation automatique des mises à jour.
✔ Design épuré et responsive.

Inconvénients :
❌ Service payant (8 $/mois).
❌ Moins personnalisable que d'autres solutions open-source.

📌 Comment l’utiliser ?

  1. Aller dans Paramètres → Core Plugins et activer Obsidian Publish.
  2. Sélectionner les notes à publier.
  3. Configurer l’URL et publier !

📌 Exemple de site :
https://publish.obsidian.md

✅ 2. Obsidian + MkDocs (Gratuit & Open-Source)

🚀 Idéal pour un site statique type documentation
Tu peux convertir tes notes Markdown en un site web statique avec MkDocs (et son thème Material pour une belle mise en page).

Avantages :
✔ Gratuit et open-source.
✔ Très personnalisable (thèmes, extensions).
✔ Compatible avec GitHub Pages, Netlify, Vercel pour l’hébergement.

Inconvénients :
❌ Configuration initiale un peu plus technique.
❌ Pas de synchronisation automatique avec Obsidian.

📌 Comment l’utiliser ?

  1. Installer MkDocs :

    pip install mkdocs-material

  2. Copier tes notes Markdown d’Obsidian dans un projet MkDocs.

  3. Générer le site statique :

    mkdocs build

  4. Héberger le site (GitHub Pages, Netlify, etc.).

📌 Exemple de site avec MkDocs :
https://squidfunk.github.io/mkdocs-material/

✅ 3. Obsidian + Quartz (Hugo)

🚀 Idéal pour un blog ou un site plus avancé
Quartz est un framework basé sur Hugo qui permet de publier un site web à partir de notes Obsidian.

Avantages :
✔ Rapide et optimisé pour le SEO.
✔ Très flexible et personnalisable.
✔ Peut être hébergé gratuitement sur GitHub Pages, Cloudflare Pages, Netlify, Vercel.

Inconvénients :
❌ Un peu plus complexe à mettre en place.

📌 Comment l’utiliser ?

  1. Cloner Quartz :

    git clone https://github.com/jackyzha0/quartz.git

  2. Ajouter tes fichiers Markdown d’Obsidian dans le dossier content/.

  3. Générer et publier le site.

📌 Exemple de site avec Quartz :
https://jackyzha0.com/

✅ 4. Obsidian + Docusaurus (Pour la Documentation Technique)

🚀 Idéal pour documenter des projets techniques et APIs
Docusaurus est un framework développé par Facebook pour créer des sites de documentation.

Avantages :
✔ Très bien adapté aux projets techniques et à la documentation.
✔ Permet d’ajouter des recherches, des mises en page avancées, et du versioning.
✔ Hébergement facile sur GitHub Pages, Netlify, Vercel.

Inconvénients :
❌ Plus complexe à configurer pour un site personnel.

📌 Comment l’utiliser ?

  1. Installer Docusaurus :

    npx create-docusaurus@latest my-site classic

  2. Copier tes notes Markdown d’Obsidian dans le dossier docs/.

  3. Démarrer le serveur :

    npm run start

📌 Exemple de site avec Docusaurus :
https://docusaurus.io/

🎯 Quelle solution choisir ?

Solution Facilité d'utilisation Personnalisation Hébergement Prix
Obsidian Publish ⭐⭐⭐⭐⭐ ⭐⭐ Hébergé par Obsidian Payant
MkDocs ⭐⭐⭐ ⭐⭐⭐⭐⭐ GitHub Pages, Netlify, etc. Gratuit
Quartz (Hugo) ⭐⭐⭐ ⭐⭐⭐⭐ GitHub Pages, Vercel, etc. Gratuit

Si tu cherches la solution la plus simple, Obsidian Publish est idéal, mais payant.
Si tu veux une solution gratuite et open-source, MkDocs ou Quartz sont de bonnes options.
Si tu veux une documentation technique avancée, Docusaurus est le meilleur choix.

Obsidian + GitHub est une excellente solution pour assurer une sauvegarde cloud gratuite, un historique des modifications et une synchronisation multi-appareils.

🔹 Objectif : Configurer Obsidian pour synchroniser tes notes sur GitHub et y accéder depuis n’importe quel appareil.
🔹 Avantages :
✅ Gratuit (contrairement à Obsidian Sync)
✅ Versioning (historique des modifications)
✅ Accès sur plusieurs appareils

🛠️ Étape 1 : Créer un dépôt GitHub pour stocker les notes

  1. Va sur GitHub et connecte-toi.
  2. Clique sur New Repository.
  3. Donne-lui un nom (ex. Obsidian-Vault).
  4. Coche "Private" si tu veux que tes notes restent personnelles.
  5. Clique sur Create Repository.

🛠️ Étape 2 : Installer Git sur ton ordinateur

Si Git n’est pas encore installé :
🔹 Windows : Télécharge Git ici
🔹 Mac : Tape dans le terminal :

brew install git

🔹 Linux (Debian/Ubuntu) :

sudo apt install git

Une fois installé, vérifie avec :

git --version

🛠️ Étape 3 : Initialiser Git dans ton dossier Obsidian

  1. Ouvre Obsidian et trouve ton dossier "vault" (ex. Documents/Obsidian/).

  2. Ouvre un terminal (ou Git Bash sur Windows) et place-toi dans ce dossier :

    cd /chemin/vers/ton/vault

  3. Initialise Git dans ce dossier :

    git init

  4. Lie-le à ton dépôt GitHub :

    git remote add origin https://github.com/ton-utilisateur/Obsidian-Vault.git

  5. Ajoute et valide les fichiers :

    git add .
git commit -m "Initial commit"

  6. Envoie sur GitHub :

    git branch -M main
git push -u origin main

  7. Ton vault est maintenant sauvegardé sur GitHub ! 🎉

🛠️ Étape 4 : Automatiser la synchronisation avec un plugin Git dans Obsidian

🔹 1. Installer le plugin "Obsidian Git"

  • Ouvre Obsidian → Paramètres → Community Plugins.
  • Active "Allow Third-Party Plugins".
  • Cherche "Obsidian Git" et installe-le.

🔹 2. Configurer le plugin

  • Dans Obsidian Git Settings :
    • Auto-pull interval : 5 min (pour récupérer les mises à jour).
    • Auto-push interval : 10 min (pour envoyer tes modifications).
    • Active "Pull changes before pushing" (évite les conflits).

Désormais, Obsidian enregistre automatiquement tes modifications sur GitHub toutes les X minutes. 🚀

🛠️ Étape 5 : Accéder à tes notes sur un autre appareil

Sur un autre PC ou un téléphone :

  1. Installe Obsidian et Git.

  2. Clone ton dépôt avec :

    git clone https://github.com/ton-utilisateur/Obsidian-Vault.git

  3. Ouvre Obsidian et sélectionne ce dossier comme vault.

  4. Active Obsidian Git pour automatiser la synchro.

🎯 Bonus : Ignorer certains fichiers avec .gitignore

Dans ton dossier vault, crée un fichier .gitignore avec :

.obsidian/workspace
node_modules/
*.log

👉 Cela empêchera d’envoyer certains fichiers inutiles sur GitHub.

✅ Conclusion

Tu as maintenant une synchronisation cloud gratuite pour Obsidian grâce à GitHub ! 🔥
Si tu veux aller plus loin, tu peux même héberger un site web avec MkDocs ou Quartz.
