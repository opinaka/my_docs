Swagger

Swagger ou OpenAPI, est un excellent moyen de documenter vos APIs de manière interactive et dynamique, et ce, pour différents langages de programmation.

Pourquoi utiliser Swagger ?

  • Documentation interactive : Swagger UI offre une interface conviviale pour tester vos APIs directement depuis un navigateur.
  • Génération de code : Swagger Codegen vous permet de générer des clients et serveurs dans de nombreux langages.
  • Conception collaborative : Swagger Editor permet de concevoir et valider les APIs avec une interface graphique.

1. Swagger avec Python (Flask)

Installation avec Flask et Flask-RESTPlus

  1. Installer Flask et Flask-RESTPlus : Pour intégrer Swagger dans une application Flask, vous pouvez utiliser la bibliothèque Flask-RESTPlus.

    pip install flask flask-restplus

  2. Créer un fichier app.py avec la configuration Swagger :

    from flask import Flask
from flask_restplus import Api, Resource

app = Flask(__name__)
api = Api(app)

@api.route('/hello')
class HelloWorld(Resource):
    def get(self):
        """
        Cette méthode retourne un message de bienvenue.
        ---
        responses:
          200:
            description: Un message de bienvenue
            examples:
              application/json: {"message": "Bonjour, Swagger!"}
        """
        return {"message": "Bonjour, Swagger!"}

if __name__ == '__main__':
    app.run(debug=True)

  3. Lancer l'application et accéder à la documentation Swagger via http://localhost:5000/ :

    python app.py

Explication

  • La classe Api est utilisée pour initialiser Swagger.
  • La méthode get() de la classe HelloWorld est annotée avec des commentaires Doxygen-esque permettant à Swagger de générer la documentation.

2. Swagger avec JavaScript (Express.js)

Installation avec Express.js et Swagger-jsdoc

  1. Installer Express et Swagger-jsdoc :

    npm install express swagger-jsdoc swagger-ui-express

  2. Créer un fichier app.js avec Swagger :

    const express = require('express');
const swaggerJsdoc = require('swagger-jsdoc');
const swaggerUi = require('swagger-ui-express');

const app = express();

// Configuration de Swagger
const options = {
  definition: {
    openapi: '3.0.0',
    info: {
      title: 'API de test',
      version: '1.0.0',
      description: 'API simple pour tester Swagger avec Express.js',
    },
  },
  apis: ['./app.js'], // Répertoire de vos fichiers où Swagger recherchera les annotations
};

const specs = swaggerJsdoc(options);

// Utilisation de Swagger UI pour exposer la documentation
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(specs));

// Définir un endpoint avec des annotations Swagger
/**
 * @swagger
 * /hello:
 *   get:
 *     description: Retourne un message de bienvenue
 *     responses:
 *       200:
 *         description: Un message de bienvenue
 *         content:
 *           application/json:
 *             example:
 *               message: "Bonjour, Swagger!"
 */
app.get('/hello', (req, res) => {
  res.json({ message: 'Bonjour, Swagger!' });
});

app.listen(3000, () => {
  console.log('API running on http://localhost:3000');
});

  3. Lancer l'application et accédez à la documentation Swagger à http://localhost:3000/api-docs.

3. Swagger avec TypeScript (Express.js)

Installation avec TypeScript et Swagger-jsdoc

  1. Installer les dépendances nécessaires :

    npm install express swagger-jsdoc swagger-ui-express
npm install --save-dev typescript @types/node @types/express

  2. Créer un fichier app.ts avec Swagger :

    import express from 'express';
import swaggerJsdoc from 'swagger-jsdoc';
import swaggerUi from 'swagger-ui-express';

const app = express();

// Swagger Options
const options = {
  definition: {
    openapi: '3.0.0',
    info: {
      title: 'API de test avec TypeScript',
      version: '1.0.0',
      description: 'API simple pour tester Swagger avec Express.js et TypeScript',
    },
  },
  apis: ['./app.ts'], // Fichiers à scanner pour Swagger
};

const specs = swaggerJsdoc(options);

// Utilisation de Swagger UI
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(specs));

// Exemple d'API avec annotations Swagger
/**
 * @swagger
 * /hello:
 *   get:
 *     description: Retourne un message de bienvenue
 *     responses:
 *       200:
 *         description: Un message de bienvenue
 *         content:
 *           application/json:
 *             example:
 *               message: "Bonjour, Swagger!"
 */
app.get('/hello', (req, res) => {
  res.json({ message: 'Bonjour, Swagger!' });
});

app.listen(3000, () => {
  console.log('API running on http://localhost:3000');
});

  3. Compiler le code TypeScript et démarrer l'application :

    • Compiler TypeScript : 
      tsc app.ts
    • Lancer l'application compilée : 
      node app.js

  4. Accéder à la documentation Swagger à l'URL http://localhost:3000/api-docs.

Bonnes pratiques pour la documentation avec Swagger

  1. Utiliser des annotations claires et structurées : Décrivez chaque endpoint avec des détails sur les paramètres, les réponses et les erreurs possibles.
  2. Fournir des exemples : Ajoutez des exemples concrets dans la documentation des réponses pour aider les utilisateurs à comprendre comment interagir avec votre API.
  3. Sécurité : Documentez les mécanismes de sécurité de votre API, tels que l'authentification avec JWT ou OAuth2.
  4. Validation continue : Utilisez un outil comme Swagger Editor pour valider et tester votre spécification OpenAPI en temps réel.

Conclusion

Swagger est un excellent outil pour la documentation d'APIs RESTful. En l'intégrant à des projets Python, JavaScript ou TypeScript, vous pouvez facilement générer une documentation interactive et tester vos endpoints en temps réel. Pour plus d'informations, consultez la documentation officielle de Swagger.

Vous pouvez également consulter des vidéos d'introduction pour Swagger sur YouTube.
