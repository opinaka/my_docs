Swagger UI et ReDoc avec FastAPI, Flask, JavaScript et TypeScript

Voici un tutoriel complet sur l'intégration de Swagger UI et ReDoc avec FastAPI, Flask, JavaScript et TypeScript. Ce tutoriel couvrira les étapes pour générer la documentation API à l'aide de Swagger UI et ReDoc en utilisant ces différents environnements.

Prérequis :

  • Une spécification OpenAPI (fichier YAML ou JSON) pour décrire votre API.
  • Un serveur local ou un environnement de développement pour tester votre API.

1. Swagger UI et ReDoc avec FastAPI (Python)

Installation et configuration

FastAPI est déjà conçu pour générer automatiquement la documentation Swagger (Swagger UI) à partir des annotations dans le code. ReDoc est également intégré par défaut.

  1. Installer FastAPI et Uvicorn :

    pip install fastapi uvicorn

  2. Créer un fichier main.py avec FastAPI :

    from fastapi import FastAPI

app = FastAPI()

@app.get("/hello")
def read_hello():
    """
    Cette API renvoie un message de bienvenue.
    ---
    responses:
      200:
        description: Un message de bienvenue
        content:
          application/json:
            example:
              message: "Bonjour, FastAPI et Swagger!"
    """
    return {"message": "Bonjour, FastAPI et Swagger!"}

  3. Lancer l'application avec Uvicorn :

    uvicorn main:app --reload

  4. Accéder à la documentation :

    • Swagger UI : http://127.0.0.1:8000/docs
    • ReDoc : http://127.0.0.1:8000/redoc

    FastAPI fournit automatiquement la documentation Swagger et ReDoc basée sur votre code. Il n'y a pas besoin de configurations supplémentaires.

2. Swagger UI et ReDoc avec Flask (Python)

Installation et configuration

  1. Installer Flask et Flask-RESTPlus pour Swagger UI :

    pip install flask flask-restplus

  2. Créer un fichier app.py avec Flask et Flask-RESTPlus :

    from flask import Flask
from flask_restplus import Api, Resource

app = Flask(__name__)
api = Api(app)

@api.route('/hello')
class HelloWorld(Resource):
    def get(self):
        """
        Cette API renvoie un message de bienvenue.
        ---
        responses:
          200:
            description: Un message de bienvenue
            content:
              application/json:
                example:
                  message: "Bonjour, Flask et Swagger!"
        """
        return {"message": "Bonjour, Flask et Swagger!"}

if __name__ == '__main__':
    app.run(debug=True)

  3. Lancer l'application Flask :

    python app.py

  4. Accéder à la documentation Swagger :

    • Swagger UI : http://127.0.0.1:5000/
    • ReDoc : Vous pouvez ajouter ReDoc en utilisant un serveur ou une version locale.

    Note : Flask-RESTPlus permet d'utiliser Swagger UI pour générer automatiquement la documentation. ReDoc n'est pas intégré par défaut, mais vous pouvez ajouter ReDoc avec une solution comme redoc-cli ou un serveur supplémentaire.

3. Swagger UI et ReDoc avec JavaScript (Express.js)

Installation et configuration

  1. Installer Express et Swagger-jsdoc :

    npm install express swagger-jsdoc swagger-ui-express

  2. Créer un fichier app.js avec Express et Swagger-jsdoc :

    const express = require('express');
const swaggerJsdoc = require('swagger-jsdoc');
const swaggerUi = require('swagger-ui-express');

const app = express();

const options = {
  definition: {
    openapi: '3.0.0',
    info: {
      title: 'API avec Express',
      version: '1.0.0',
      description: 'Documentation d\'une API avec Express et Swagger',
    },
  },
  apis: ['./app.js'], // Le fichier contenant les annotations
};

const specs = swaggerJsdoc(options);

// Documentation Swagger UI
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(specs));

/**
 * @swagger
 * /hello:
 *   get:
 *     description: Retourne un message de bienvenue
 *     responses:
 *       200:
 *         description: Un message de bienvenue
 *         content:
 *           application/json:
 *             example:
 *               message: "Bonjour, Express et Swagger!"
 */
app.get('/hello', (req, res) => {
  res.json({ message: 'Bonjour, Express et Swagger!' });
});

app.listen(3000, () => {
  console.log('API running on http://localhost:3000');
});

  3. Lancer l'application Express :

    node app.js

  4. Accéder à la documentation Swagger UI à http://localhost:3000/api-docs.

Ajouter ReDoc :

Pour ajouter ReDoc à votre serveur Express, vous pouvez utiliser un fichier HTML statique généré via redoc-cli ou ajouter un serveur statique dans votre application Express.

  1. Installer redoc-cli :

    npm install -g redoc-cli

  2. Générer le fichier HTML de la documentation :

    redoc-cli bundle swagger.yaml

  3. Servir ReDoc sur votre serveur Express :

    Vous pouvez héberger le fichier généré en tant que fichier statique dans votre serveur Express ou utiliser ReDoc en ligne pour afficher la documentation.

4. Swagger UI et ReDoc avec TypeScript (Express.js)

Installation et configuration

  1. Installer les dépendances nécessaires :

    npm install express swagger-jsdoc swagger-ui-express
npm install --save-dev typescript @types/node @types/express

  2. Créer un fichier app.ts avec Express et Swagger-jsdoc :

    import express from 'express';
import swaggerJsdoc from 'swagger-jsdoc';
import swaggerUi from 'swagger-ui-express';

const app = express();

const options = {
  definition: {
    openapi: '3.0.0',
    info: {
      title: 'API avec TypeScript',
      version: '1.0.0',
      description: 'Documentation d\'une API avec Express et Swagger',
    },
  },
  apis: ['./app.ts'], // Le fichier contenant les annotations
};

const specs = swaggerJsdoc(options);

// Swagger UI
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(specs));

/**
 * @swagger
 * /hello:
 *   get:
 *     description: Retourne un message de bienvenue
 *     responses:
 *       200:
 *         description: Un message de bienvenue
 *         content:
 *           application/json:
 *             example:
 *               message: "Bonjour, TypeScript et Swagger!"
 */
app.get('/hello', (req, res) => {
  res.json({ message: 'Bonjour, TypeScript et Swagger!' });
});

app.listen(3000, () => {
  console.log('API running on http://localhost:3000');
});

  3. Compiler et exécuter l'application TypeScript :

    • Compiler le fichier TypeScript : 
      tsc app.ts
    • Lancer l'application compilée : 
      node app.js

  4. Accéder à la documentation Swagger UI à http://localhost:3000/api-docs.

Conclusion

  • FastAPI : La documentation Swagger et ReDoc est intégrée automatiquement. Vous n'avez rien à configurer de plus pour afficher la documentation.
  • Flask : Vous pouvez utiliser Flask-RESTPlus pour intégrer Swagger UI, mais ReDoc nécessitera une installation ou un service externe.
  • Express (JavaScript et TypeScript) : Utilisez swagger-jsdoc pour générer la documentation à partir de votre code, puis servez-la avec Swagger UI. Vous pouvez également intégrer ReDoc via un fichier HTML statique ou en ligne.

Cela vous permet de documenter vos APIs de manière robuste et interactive avec Swagger UI et ReDoc dans différents environnements.
